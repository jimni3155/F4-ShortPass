# 🎯 Data Venture 결선 대비 서비스 방향 정리 (Team F4)

## 1. 프로젝트 개요

**주제:** 불필요한 채용 과정을 줄이고, AI 면접을 통해 채용 절차를 효율화하는 B2B 서비스  
**핵심 목표:**

- 기업 HR이 **AI 면접 평가를 신뢰할 수 있는 구조** 구축
- 지원자는 **객관적인 역량 검증**과 **맞춤형 기업 추천**을 경험

---

## 2. 우리가 지향해야 하는 것

### ✅ 기업 중심 (B2B 우선)

- **주 타깃:** HR 담당자, 채용 실무자
- **핵심 가치:** “AI 면접 결과를 즉시 보고서 형태로 활용 가능”
- 기업이 체감할 수 있는 포인트:
  - 면접 시간 절감
  - 인재 검증 과정의 객관화
  - 데이터 기반 채용 의사결정

### ✅ 신뢰 가능한 AI 평가

- “AI가 왜 이렇게 판단했는가?”에 대한 **명확한 근거 제공**
- 평가 기준이 고정이 아니라 **기업 인재상 기반 루브릭 구조**
- 평가 결과를 수치화하되, **근거 문장·키워드 시각화**

### ✅ 시각적 전달력

- 심사위원·기업 모두가 **AI 판단 구조를 직관적으로 이해**하도록 시각화
- “AI 판단 → 근거 → 결과 활용”의 흐름이 한눈에 보이게 구성

---

## 3. 우리가 고려해야 하는 것

### ⚙️ 서비스 구조적 고려

1. **기업 1:1 세션 구조로 변경**

   - 기존 다대일(여러 기업 → 1 지원자) 구조는 해석 난이도 높음
   - → 한 기업 내 여러 페르소나가 순차적으로 질문하는 구조로 단순화

2. **페르소나 유지 및 재설계**

   - 기업 내부 면접관 유형(기술형, 논리형, 컬처형 등)을 **패널 형태로 유지**
   - 각 페르소나가 다른 관점에서 질문 및 평가 수행
   - 최종 리포트에서 페르소나별 점수·근거 분리 표시

3. **AI 평가 루브릭 정립**
   - 평가 항목은 최소 4개로 고정:
     - 논리성(Logical Thinking)
     - 문제해결력(Problem Solving)
     - 협업력(Teamwork)
     - 성장 가능성(Growth Potential)
   - 모든 점수는 해당 루브릭 기반으로 계산되어야 함

---

### 🧠 기술적 고려

- **AI 꼬리질문 생성 로직**
  - 기업 인재상 + 지원자 답변 + 루브릭 약점항목 기반
  - 약점 보완형 질문 생성 (예: “그때 갈등 상황은 어떻게 조율했나요?”)
- **점수화 및 근거 로깅**
  - LLM 점수 산출 시 `explanation` 필드에 평가 근거 저장
  - `log_json`으로 coherence, evidence_match 등 보조 지표 기록
- **모델 중립화 파이프라인**
  - 기업·직무별 점수 편차 모니터링 (PSI/KS)
  - 주기적 기준선 재설정(z-score normalization)
  - 모델 업데이트 로그 표시(예: “최근 학습일: 2025.11.09”)

---

### 💬 발표/UX 측면 고려

1. **심사위원 관점에서 보여야 할 키 포인트**

   - “AI 판단 근거가 눈에 보인다”
   - “기업이 실제로 활용 가능한 구조다”
   - “AI 평가 결과가 신뢰할 수 있다”

2. **UI 구조 방향**

   - 최종 리포트 화면에 다음 순서로 구성:
     1. 총점 및 등급 요약 (Fit Score + 인재상 요약)
     2. 루브릭별 세부 점수표 (Radar Chart)
     3. 근거 문장 / AI 판단 로그
     4. 기업별 매칭 그래프 (유사도 기반)
     5. 기업 피드백 및 PDF 보고서 버튼

3. **발표 메시지**
   > “저희 서비스는 단순히 면접을 자동화하는 것이 아니라,  
   > AI가 평가하는 과정을 기업이 **납득할 수 있는 근거 중심 구조로 시각화**한  
   > **설명 가능한(Explainable) AI 채용 평가 솔루션**입니다.”
